Boolean Type

       - Boolean is used in decision making. 

       - HTML provides various boolean attributes for elements, like          readonly, required, checked, selected, disabled, open etc.

       - JavaScript uses them as properties for elements.



       FAQ: What is difference between attribute & property?

       Ans:  Attribute is immutable, it has a fixed value. 

               Property is mutable, it can change dynamically. 



       - Boolean properties are controlled by using “true” or “false” as values. 

       - JavaScript can handle boolean true with 1 and false with 0, but it is not recommended. 



       Syntax:

                 var stock = true;

                 if(stock==1)                // verifying true : not recommended 

                 {

                 }

   

                 if(stock==true)            // recommended 

                 {

                 }



       - Boolean values are compared by using decision making operators.



               ==                Equal

               ===            Identical Equal / Strict Equal 

               !=                Not Equal

               !==            Not Identical

               >                Greater than

               >=                Greater than or equal

               <                Lesser than

               <=                Lesser than or equal

               

               &&            Logical AND

               ||                Logical OR

               !                NOT



               ?:                Ternary Operator





       FAQ: What is difference between “==“ & “===“ ?

       Ans:   “==“ can compare values of two different data types.

                “===“ can compare only values of same data types.



                Syntax:

                       10 == “10”           // true 

                       10 === “10”         // false  



                “===“ will always return false if it is comparing different data types.

Undefined Type:

       - It is a type defined for reference without value. 

       - If value is not defined into any reference variable then it is set with “undefined” while

         compiling or interpreting. 

       - You can verify by using “undefined” keyword. 



Syntax:

           var  price;

           if(price===undefined) { }                   // not good

           if(price!==undefined) { }                // not good

           if(price){ }                                      // Good : It returns true if value is defined. 



Ex:

<script>

     var name = "Samsung TV";

     var price;

    

     if(price)

     {

          document.write(`Name=${name}<br>Price=${price}`);

     } else {

         document.write(`Name=${name}`);

     }

</script>





        FAQ: What is difference between undefined and not-defined?

        Ans:  Undefined verifies the value, not-defined verifies the reference.



Null Type:

       - It refers to a value at runtime of application. 

       - If value is not defined at runtime then it returns “null”. 

       - You can verify by using “null” keyword. 



Ex:

<script>

     var name = "Samsung TV";

     var price = prompt("Enter Price");

    

     if(price!==null)

     {

          document.write(`Name=${name}<br>Price=${price}`);

     } else {

         document.write(`Name=${name}`);

     }

</script>
