CSS Display

   1. none                : It hides the element and removes allocated space. 
   
FAQ: What is difference between “display:none” & “visibility:hidden” ?
Ans:   Visibility-hidden can hide the element but keeps the allocated space.
         Display-none will hide and remove the allocated space. 
   
       Syntax:
               h1 {
                    display: none;
                           (or) 
                    visibility: hidden;
               }

   2. inline            : It allows to display any content beside the element. 

   3. block                : It keeps adjacent elements in the next line. It will not allow any content
                             beside the element. 

   4. inline-block    : It keeps adjacent elements in the same line and also allows to control
                             the block attributes like width & height.

Ex:
<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Document</title>
   <link rel="stylesheet" href="../node_modules/bootstrap-icons/font/bootstrap-icons.css">
   <style>
      h1 {
         display: inline;
      }
      span {
        border:1px solid red;
        padding: 5px;
        width: 150px;
        display: inline-block;
      }
   </style>
</head>
<body>
    <h1>Welcome</h1>
    to CSS.
    <br><br>
    <span>Welcome</span> to JavaScript
</body>
</html>

   5. flex            : It is used for a container to keep its elements flexible. 
                         It can change the display orientation of elements in container to row or column
                        
   6. flex-direction : It can have the values
                               a) row
                               b) column
                               c) row-reverse
                               d) column-reverse 

   7. flex-wrap        : It is used to wrap elements inside flex container. It can have values
                                    a) wrap
                                  b) no-wrap
                             Wrapping is technique of allowing elements to next line when they reach
                             the right margin.

   Note:  The default is “no-wrap” which is used to shrink the contents inside container.
              It may span contents outside container if there is no sufficient space, which is not
              good for responsive design.

       8. justify-content    : It allows to justify the contents inside container with following values                                        a) baseline   [ left in row, top in column ]
                                       b) end          [ right in row, bottom in column ]
                                       c) center      
                                       d) space-between
                                       e) space-around
                                       f) space-evenly

       9. align-items            :  If direction is row you can align item vertically top, center, bottom. 
                                      If direction is column you can align items horizontally left, center,
                                      right. 
                                       a) baseline
                                       b) end
                                       c) center

Ex:

<html lang="en">

           10. grid                : It is used to create a responsive layout. 
                                     A responsive layout allows to change content dynamically.
                                     You can rearrange contents into rows columns. 
       
           11. grid-template-columns : It is used to configure a set of columns for grid.
                                                    Columns are defined using “fractions” [ 12fr ] or
                                                    1200px. 
           12. grid-row        : It can set elements in specified row.

           13. grid-column : It can set elements into specified column. 

           14. grid-gap        : It sets space between columns. 

           Note: To create a responsive layout you need media queries to configure.       
Ex:
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        header, nav, main, aside, footer {
            border:1px solid black;
            padding: 10px;
            margin: 5px;
            font-size: 20px;
        }
        main {
            height: 400px;
        }
        @media screen and (orientation: landscape)
        {
            body {
                display: grid;
                grid-template-columns: 2fr 8fr 2fr;
            }
            header {
                grid-row: 1;
                grid-column: 1/4;
            }
            nav {
                grid-row: 2;
                grid-column: 1;
            }
            main {
                grid-row: 2;
                grid-column: 2;
            }
            aside {
                grid-row: 2;
                grid-column: 3;
            }
            footer {
                grid-row: 3;
                grid-column: 1/4;
            }
        }
        @media screen and (orientation: portrait)
        {
            body {
                display: grid;
                grid-template-columns: 12fr;
            }
            header {
                grid-row: 1;
            }
            nav {
                grid-row : 2;
            }
            main {
                grid-row: 3;
            }
            aside {
                grid-row: 4;
               
            }
            footer {
                grid-row: 5;
            }
        }
    </style>
</head>
<body>
    <header>
        Header
    </header>
    <nav>Nav</nav>
    <main>Main</main>
    <aside>Aside</aside>
    <footer>Footer</footer>
</body>
</html>
                        
   Note:  “grid-columns” can be merged by using “1/n”, which is from 1 up to “n”.

               {
                grid-column: 1/4;         // merges 3 columns 
               }
