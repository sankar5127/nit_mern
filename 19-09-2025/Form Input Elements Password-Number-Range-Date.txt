Regular Expression

- Meta Characters

- Quantifiers



           (?=.*[A-Z])                    At least one uppercase letter is required

           (?=.*[0-9])                    At least on numeric digit required

           (?=.*[!@#$%])            At least on special character. 

       

Syntax: User Name 4 to 15 chars with at least one uppercase letter and one numeric value.



                       pattern=“(?=.*[A-Z])(?=.*[0-9])\w{4,15}”



           Write a pattern for user name 4 to 10 chars starting with uppercase letter.

           It can have alphabet, number or special char, but must start with uppercase alphabet.



                       pattern=“[A-Z]\w{4,9}”



                                                       Password



 - It a string input masked with password character “*”. 

 - All attributes same as textbox. 



Syntax:

           <input type=“password”  id=“”  name=“”  class=“” required  pattern=“”>



- It is not recommended to use “list” attribute.



                                                        Number

- It allows only numeric type of value. 

- Number attributes include

       a) min

       b) max

       c) step



Syntax:

       <input type=“number”  name=“age” min=“15” max=“30” step=“1” value=“22”>

       <input type=“number”  name=“rate” min=“10.45” max=“19.45” step=“0.01” value=“11”>



                                                      Range



 - It is similar to number but provides a slider to select value.

 - Attributes same as number

   a) min

   b) max

   c) step



Syntax:

   <input type=“range”  min=“1” max=“5” value=“1” step=“1”>



                                                     Email

 - It is similar to text but validates email address by verifying “@“ character. 

 - You can use text attributes like required, minlength, value, readonly etc.



Syntax:

   <input type=“email” name=“email” required>



                                                     URL

 - It is a string that validates URL.

 - A valid URL comprises of proper protocol and domain. 



                   www.amazon.in               // invalid

                   https://www.amazon.in  // valid 

 

                                                       Color

- It shows a color picker that allows to select any color. 

- The color default value must be a “6” chars hexadecimal code with “#”.



Syntax:

       <input type=“color” name=“color” value=“#ff00ff”>



                                                     Date & Time



 - HTML provides various types to handle date & time values.

       a) date

       b) time

       c) datetime-local 

       d) week

       e) month

       f) year

 - You can’t select a range of date values. 



Syntax:

       <input type=“date | time | week ..”> 



 - You can restrict the date using “min” & “max”.

 - For developer date is always “yy-mm-dd” format. 



Syntax:

       <input type=“date”  min=“2025-09-19” max=“2025-10-09”   value=“2025-09-22”>



                                                         File

 - It allows to browse and select a file from client device. 

 - It is not a file uploader, it is just a file browser.



Syntax:

       <input type=“file” name=“photo”>



 - You can’t restrict the file type but can set filters for file type by using “accept” attribute.



Syntax:

       <input type=“file” accept=“.jpg, .png” name=“photo”>



 - It allows to select only one file, you can set “multiple” attribute to allow selection of multiple files.



Syntax:

       <input type=“file”  accept=“.jpg”  name=“photos”  multiple>
