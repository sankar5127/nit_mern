3. document.write()

       - It presents output in a new screen of same page.

       - It can render value, expression result & markup. 



Syntax:

       document.write(“value | expression | markup”);



       - Line breaks are defined using markup. 



Ex:

<!DOCTYPE html>

<html lang="en">

<head>

    <meta charset="UTF-8">

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

        function DeleteClick(){

            result = confirm("Delete Record\nAre you sure?");

            if(result==true){

                document.write("<h3><font color='red'>Deleted..</font></h3><a href='output.html'>Back</a>");

            } else {

                alert("Canceled..");

            }

            

        }

    </script>

4. innerText  & textContent 

       - innerText is ES4 output technique of rendering result into a container that can display

         text. 

       - textContent is from E5+ version and recommended for both semantic and non-semantic

         elements.

       - Both can’t display complex formats of text. [ both are RC type ].

   

Syntax:

           document.querySelector(“div”).textContent = “value | expression result”;

           document.querySelector(“p”).innerText = “value | expression result”;



Ex:

<!DOCTYPE html>

                document.querySelector("p").innerText = "Canceled..";

            }

            

        }

    </script>

</head>

<body>

    <button onclick="DeleteClick()">Delete</button>

    <p></p> 

</body>

</html>



5. innerHTML

       - The target element must allow markup. 

       - It renders the resulting markup into target element. 



Syntax:

           document.querySelector(“p”).innerHTML = “<h2> Welcome </h2>”;

6. outerHTML 

       - It is similar to innerHTML but replaces the target element with new element. 

       - It can render a markup by replacing existing markup. 



Syntax:

           document.querySelector(“p”).outerHTML = “<h2> Welcome </h2>”;



           <h2>Welcome</h2>

7. Console Methods

           - Console is a CLI ( Command Line Tool ) of browser.

           - It is a bebugging tool of browser. 

           - It is used by developers to test various commands in browser. 

           - JavaScript provides several contextual console methods to display message inside

             console. 



           console.log()

           console.error()

           console.warn()

           console.debug()

           console.info() etc..



           - Console methods are RC type. You can’t render markup. 

           - Line break is defined using “\n”.



Syntax:

           console.log(“value | expression result”);

           console.log(“line-1 \n line-2”);

Query String:

   - Query String is appended into URL and stored in browser address bar. 

   

                      page.html ?key=value&key=value&key=value..

   

   - It is appended using “?”, and it is a key and value collection. 

   - Multiple keys are appended using “&”. 

   - You can access query string by using browser “location.search” property. 

   - Location is a browser object and “search” is a property. 

   - Query string is returned as “string”.



                   ‘?key=value&key=value’

       

   - You can separate the string into a key & value collection by using JavaScript browser

     method “URLSearchParams()”.

   

                   name = new URLSearchParams(location.search);



   - “new” is dynamic memory allocation operator. 

   - You can get the value with reference of key.



                   name.get(“key”);
