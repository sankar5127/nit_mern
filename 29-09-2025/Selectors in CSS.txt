1. Primary Selectors

   

a) Type Selector

       - It refers to the element token name [tag].

       - It will apply styles for every occurrence of element in page. 

       - You can’t ignore for any specific occurrence.



Syntax:

           h1, div, p, table, td {



           }



b) ID Selector

       - You can configure a set of styles and store in ID reference with “#”. 

       - So that you can reuse the styles for various elements.

       - You can choose and apply only for required element. 



Syntax:    

           #name {



           }



           <h2 id=“name”>

           <p id=“name”>

           <img id=“name”>



       - ID reference is applied at 1st priority. 

       - Every element can have only one ID reference. 

       - You can’t apply multiple categories of styles for one element.



c) Class Selector



       - You can configure a set of styles under a class reference. 

       - So that you can reuse the styles for various elements.

       - It allows to configure multiple sets of styles for one element.

       - It is defined using “.” And reference name.

       - It is accessed using “class” attribute. 



Syntax:

           .one {



           }

           .two {

           

           }

           

           <div class=“one two”>

           <p class=“one”>

           <span class=“two”>

   

       - To configure multiple classes a “blank space” delimiter is used. 



Ex:

<!DOCTYPE html>

<html lang="en">

<head>

   <meta charset="UTF-8">

   <meta name="viewport" content="width=device-width, initial-scale=1.0">

   <title>Document</title>

   <style>

      .text-style {

         color:red;

         text-align: center;

      }

      .bg-style {

         background-color: yellow;

      }

      .box-style {

         border:2px double red;

         padding: 10px;

      }

   </style>

</head>

<body>

   <h1 class="box-style text-style bg-style"> Web Technologies </h1>

   <p class="box-style bg-style">Lorem ipsum dolor sit, amet consectetur adipisicing elit. Iste laudantium quibusdam, repellat asperiores accusantium, officiis et blanditiis distinctio laboriosam nostrum ipsum, nihil corporis? Sapiente quis, dicta maiores aperiam laboriosam odit!</p>

</body>

</html>



FAQ’s: 

   1. If we configure styles for elements using type, id & class then which set will apply?

   A. If different style attributes are used then all will apply.

       If same style is defined in all selectors then it uses priority.

               a) 1st      ID Selector

               b) 2nd     Class Selector

               c) 3rd     Type Selector



   2. If same class or id is defined in external stylesheet and embedded style then which set will apply?

   A. The latest will apply. [LIFO]



Note: You can group the selectors by using “,” delimiter, when you want same set of 

         styles in every reference.



Syntax:

               .className, #id_ref, type {



               }

Ex:

<!DOCTYPE html>

<html lang="en">

<head>

2. Relational Selectors 



           parent  child { }                    child selector

           parent > child { }                direct child selector

           elementA + elementB { }        Adjacent selector [ Adjacent Sibling ]

           elementA ~ elementB { }        General sibling



Syntax:

       header > span { }                It will select only to the span that is directly configured in 

                                               header.

       header span { }                    It will select all span elements inside header at any level

                                               of hierarchy.



Ex:

<!DOCTYPE html>

<html lang="en">

<head>

   <meta charset="UTF-8">

   <meta name="viewport" content="width=device-width, initial-scale=1.0">

   <title>Document</title>

   <style>

      header > span {

         color:red;

         font-weight: bold;

         font-family: Arial;

         font-size: 24px;

      }

   </style>

</head>

<body>

    <header>

        <span> Amazon </span>

        <nav>

           <span>Electronics</span>

           <span>Fashion</span>

           <span>Footwear</span>

           <span>Accessories</span>

        </nav>

    </header>

</body>

</html>

3. Structural Classes [Pseudo Classes]



       :first-child

       :last-child

       :nth-child()

       :nth-of-type()

       :nth-last-of-type()



Syntax:

           first-child                => first child element            last-child                => last child element

           nth-child(n)            => n refers to level number starting from 1

           nth-child(odd/even) => refers to even and odd occurrences 

           nth-of-type(2n)         => every 2nd occurrence starting form top or left. [starts with 2]

           nth-of-type(2n+1)    => every 2nd occurrence starting from top 1. Or left 1. 

           nth-last-of-type(2n+1) => every 2nd occurrence starting from bottom or right 1st.



Ex:

<!DOCTYPE html>

<head>

   <meta charset="UTF-8">

   <meta name="viewport" content="width=device-width, initial-scale=1.0">
